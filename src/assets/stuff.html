CSS Positioning Examples

Static Positioning (Default)

Static
Another Box

The static box is positioned according to the normal flow of the document. You cannot use top, right, bottom, left, or z-index properties to affect its position.

Relative Positioning

Static Box
Relative

The relative box is positioned relative to its normal position. Setting top, right, bottom, and left properties will shift the element away from its normal position, but it will not affect the positioning of other elements around it. The space it would have occupied in the normal flow is preserved.

Absolute Positioning

Static Box
Absolute

The absolute box is positioned relative to the nearest positioned ancestor (an ancestor with any position value other than static). If an absolute positioned element has no positioned ancestors, it uses the document body as its reference. Absolute positioning removes the element from the normal document flow, and other elements will behave as if the absolutely positioned element is not there.

Fixed

The fixed box is positioned relative to the viewport (the browser window). It will always stay in the same place even if the page is scrolled. Top, right, bottom, and left properties are used to position the element.

The sticky box is positioned based on the user's scroll position. It behaves like relative within its parent until the element is scrolled to a specified offset from the viewport. At that point, it becomes "fixed" until it reaches the boundary of its parent element.
